## Documentation Rule

When a change affects user behavior, technical architecture, operations, verification, setup, or deployment, update the relevant docs in the same piece of work.

At minimum, keep these aligned:

- `docs/user/*` for user-facing workflow or UI changes
- `docs/technical/*` for architecture, API flow, or implementation model changes
- `docs/operations/*` for deployment, config, monitoring, or runbook changes
- `docs/README.md` when the documentation structure changes
- `README.md` when the main project entry points or verification guidance change
